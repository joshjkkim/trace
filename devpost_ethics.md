Privacy: The SDK captures prompts and outputs, which can contain sensitive user data. We addressed this by isolating all data per project behind API keys, giving teams full control over what they instrument, and never using ingested data for cross-project analytics or model training.

Social impact: Cernova is net-positive by design — AI systems fail silently, and that silence hurts end users. By surfacing hallucinations, format violations, and unexpected behavior in real time, we make it easier for developers to catch problems before they reach production. The anomaly detection layer exists specifically to make AI behavior more accountable and observable.

Environmental impact: Token and cost tracking creates a natural incentive to optimize. When developers can see exactly how many tokens each step uses and what it costs, they write tighter prompts and cut waste. The anomaly engine also flags token bloat explicitly (classify steps returning paragraphs instead of a single word, JSON steps with implausibly large outputs) — pushing toward more efficient model usage.
